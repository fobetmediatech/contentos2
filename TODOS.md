## 🟠 Layer 1 — Data & Key Infrastructure

Scripts-first: these modules must run in Node.js test scripts _and_ the browser. The binary gate (`scripts/test-scraper.mjs`) must return exit code 0 before Layer 2 begins.

- [ ] **TE9** · `src/lib/storage.ts` — cross-runtime storage adapter. Returns `window.localStorage` in browser, an in-memory `Map` in Node. All other Layer 1 modules import storage from here (never `window.localStorage` directly).
- [ ] **TE2** · `src/lib/keyRotator.ts` — round-robin Apify key selection with cooldown tracking. Uses storage adapter. `isReady()` returns true only when Gemini key exists AND at least one Apify key is not in cooldown. Cooldown timestamps stored as epoch milliseconds (numbers, not Date objects). Read-then-write is synchronous to prevent TOCTOU race.
- [ ] **TE1** · `src/lib/apifyClient.ts` — **2-round scrape workflow** (confirmed from real data):
  - **Round 1**: POST `apify/instagram-profile-scraper` with `{ usernames: inputHandles }` → poll → get profiles → extract `relatedHandles` from each profile
  - **Round 2**: Deduplicate + filter (remove input handles) → POST `apify/instagram-profile-scraper` with `{ usernames: candidateHandles }` → poll → get full candidate profiles
  - Total Apify actor runs: 2 (not 1). Both are async; each needs polling.
  - Input body: `{ "startUrls": [{ "url": "https://www.instagram.com/{handle}" }] }` OR `{ "usernames": ["handle1", "handle2"] }` — verify which format the actor accepts during PRE-L1-2.
- [ ] **TE13** · **P0 — Apify async polling** — `src/lib/apifyClient.ts` (extend TE1). After obtaining `runId`, poll `GET /v2/acts/{actorId}/runs/{runId}` every 2 seconds until `status === "SUCCEEDED"`, then fetch `GET /v2/datasets/{defaultDatasetId}/items`. Use `pLimit(3)` to cap concurrent actor runs. Timeout after 110 seconds (leave 10s buffer for the 120s hard limit).
- [ ] **TE5** · `src/lib/transformers.ts` — confirmed field map from real Apify output:
  - `followersCount` → followers (guard: < 100 → ER = null)
  - `latestPosts[*].likesCount` → avg likes (12 posts scraped by default)
  - `latestPosts[*].commentsCount` → avg comments
  - **ER outlier guard**: exclude posts where `commentsCount > followersCount * 0.05` before averaging (DM-bait posts like "Comment 'Details'" inflate comments; real data shows 6,335 comments on a 101K-follower account in one post)
  - `relatedProfiles[*].username` → `relatedHandles[]` (for Round 2 scraping)
  - ER formula: `(avgLikes + avgComments) / followers * 100`
  - Result type: `NormalizedProfile` (username, fullName, biography, followersCount, profilePicUrl, verified, postsCount, isBusinessAccount, avgLikes, avgComments, engagementRate, relatedHandles)
- [ ] **TE7** · `src/lib/actors.ts` — **single actor, confirmed from real data.** Export:
  ```typescript
  export const ACTORS = {
    PROFILE_SCRAPER: 'apify/instagram-profile-scraper',
  } as const
  // Input shape: { usernames: string[] }
  // Output shape: ProfileApifyRaw[] (see transformers.ts for field map)
  ```
- [ ] **TE8** · `scripts/test-scraper.mjs` — Layer 1 binary gate. Runs in Node. Tests: CORS headers reachable, actor run starts, polling reaches SUCCEEDED, dataset fetch returns valid shape, ER formula returns non-null for valid input. Exit 0 = pass, exit 1 = block Layer 2.

## 🟡 Layer 2 — AI Processing

AI modules depend on Layer 1 transformers and the COMPETITOR_CATEGORIES config. The binary gate (`scripts/test-ai.mjs`) must return exit code 0 before Layer 3 begins.

- [ ] **TE11** · `src/shared/utils/categories.js` — `COMPETITOR_CATEGORIES` config object. Keys: internal IDs. Values: `{ label: string, uiLabel: string, taxonomy: string[] }`. This is the single source of truth for both UI labels and AI taxonomy language.
- [ ] **TE10** · `src/ai/prompts.js` — inject `COMPETITOR_CATEGORIES` at runtime into prompt templates. Category names are template literals, not hardcoded strings. Changing `categories.js` changes AI output and UI labels in a single edit.
- [ ] **TE6** · `src/ai/gemini.ts` — Gemini REST API caller (no SDK). Handle 5 distinct error modes:
  - `429 RESOURCE_EXHAUSTED` → retry with exponential backoff (max 3 attempts)
  - `400 INVALID_ARGUMENT` → surface prompt error, do not retry
  - `500 INTERNAL` → surface transient error, suggest retry
  - `503 UNAVAILABLE` → surface service unavailable
  - Empty `candidates` array → SAFETY block — surface as content policy error
- [ ] **TE12** · `scripts/test-ai.mjs` — Layer 2 binary gate. Runs in Node. Tests: prompt renders with correct taxonomy, Gemini returns valid JSON shape (10 handles, categories, rationale), error modes return correct error types. Exit 0 = pass, exit 1 = block Layer 3.
